Course Overview

The purpose of this course is to introduce cadets to the concepts underlying distributed systems. Building on the foundations of algorithm implementation, data representation, web development, and basic networking, this course focuses on the principles of constructing a distributed application. Cadets study the principles, construction, and interaction of user interface, network, web server, and database components to produce an effective distributed application. Cadets will learn new tools and skills working as a team to analyze, design, and implement a system that solves a given problem. Prereqs: IT300/CS301 and CS393.

Getting Help

Please use the Piazza system to request AI and ask general questions related to the material. Students who ask good questions (and give answers) will be eligible to earn bonus points. Remember, if you have a question about something, most likely other people will too! Please do not post assignment-related code publicly on Piazza. If your code is broken or you want help debugging, please sent a private message to your instructor and/or schedule AI.

Documentation of Assistance

While you are not permitted to share code for your projects, you are encouraged to discuss strategies for solving problems, and reviewing material. That said, every assignment you hand in must be solely you or your teams work, and must include appropriate documentation. Again, you are not allowed to share code, unless explicitly stated! Electronic copying is also strictly forbidden! Please refer to the Documentation of Academic Work, The Little Brown Book, and the IT394 Assistance and Collaboration Policy for exact methods for documentation. There are no exceptions to this policy. Any assignment received that lacks documentation will receive a zero. You are all under the West Point Honor Code: act honorably!

What types of assistance is encouraged?

The following types of assistance is not only okay, but actively encouraged in IT460. Learning does not occur in a vacuum. Asking (and answering) questions is one of the best ways to learn. You should use Piazza as a general forum for asking and answering questions. Remember that students who ask good questions (and give good answers) are eligible for bonus points at the end of the semester.

1. Clarifying ambiguities or vague points in class handouts.
2. Using code from the class web pages
3. Helping others with general questions about tools or software.
4. Helping others with high-level design or debugging issues.
